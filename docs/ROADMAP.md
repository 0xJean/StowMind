# StowMind 产品路线图

面向「一键整理文件夹」类桌面应用的核心能力规划。已完成项会勾选。

## 当前进度（状态摘要）

| 阶段 | 状态 | 说明 |
|------|------|------|
| **A 信任与范围** | ✅ 已完成 | 扫描范围、排除、预览 dry-run、撤销与部分成功策略 |
| **B 可控性** | ✅ 已完成 | 单条改分类、单条「本次不移动」 |
| **C 可靠性与平台** | ✅ 已完成 | 错误归类、云盘提示；**可选备份**（移动前复制到会话子目录） |
| **D 体验打磨** | ✅ 已完成 | 整理进度、引导、快捷键 |
| **E 进阶** | 🔄 部分完成 | 规则导入/导出、**文件夹监视**、**重复检测**已做；**定时/计划任务**未做 |

**下一优先建议**：按产品需要扩展阶段 E（例如监视触发自动扫描、重复项一键删除等）。

---

## 阶段 A：信任与范围（已完成）

- [x] **部分成功不回滚**：单项失败仅记录，成功项保留（已实现）
- [x] **整理后手动撤销**：历史记录 + 整理页横幅一键撤销（已实现）
- [x] **扫描范围说明**：界面明确「默认仅当前目录下一层」；可选「包含子文件夹」递归扫描文件
- [x] **排除规则**：可配置路径/名称片段（如 `node_modules`、`.git`），扫描时跳过
- [x] **执行前预览（Dry-run）**：仅生成 `源路径 → 目标路径` 列表，不写入磁盘；确认后再执行

## 阶段 B：可控性（已完成）

- [x] **扫描结果可编辑**：单条修改目标分类（下拉选择）；执行时使用用户最终分类
- [x] **单条跳过**：勾选「本次不移动」（可选，依赖列表 UI）

## 阶段 C：可靠性与平台

- [x] **云盘占位文件**：移动/复制失败时，对 Windows 常见「仅联机」错误码（如 `362` *ERROR_CLOUD_FILE_NOT_AVAILABLE*）输出 **`[云盘]`** 提示；其他平台无专用码时走 **`[其他]`** 与系统原文
- [x] **长路径 / 占用**：整理与撤销路径上的 `io::Error` 统一归类为 **`[权限]` / `[占用]` / `[路径]` / `[云盘]` / `[冲突]` / `[其他]`**（见 `src-tauri/src/organizer.rs` 中 `classify_io_error_msg`）
- [x] **可选备份**：执行整理前复制到用户指定的备份根目录下的会话子目录（`备份根/<sessionId>/相对待整理目录的路径`）；备份根不能与待整理目录相同或位于其内部

## 阶段 D：体验打磨（已完成）

- [x] **整理阶段进度**：`move_files` / `move_folders` 每处理一项 emit `organize-progress`；前端进度条与当前路径
- [x] **首次引导**：无扫描结果且非扫描中时展示「使用流程」卡片
- [x] **快捷键**：**Ctrl+Enter**（macOS **⌘+Enter**）在非输入框焦点时触发执行整理

## 阶段 E：进阶（长期）

- [x] **文件夹监视**：设置页配置路径（每行一个）；`notify` 递归监视 + 约 2s 防抖；`watch-folder-change` 事件 + 全局 toast（**不**自动整理）
- [x] **规则导入导出**（分类模板）：设置页 **导出规则** / **导入规则**（JSON，`src/lib/categoryRules.ts`）
- [x] **重复文件检测**：**重复文件**页；先按大小分组再全文件 SHA-256；支持递归与扫描排除；`duplicate-scan-progress` 进度事件
- [ ] **定时 / 计划任务**（未做；例如按间隔提示整理或触发扫描）
