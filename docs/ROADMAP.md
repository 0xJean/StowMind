# StowMind 产品路线图

面向「一键整理文件夹」类桌面应用的核心能力规划。已完成项会勾选。

## 当前进度（状态摘要）

| 阶段 | 状态 | 说明 |
|------|------|------|
| **A 信任与范围** | ✅ 已完成 | 扫描范围、排除、预览 dry-run、撤销与部分成功策略 |
| **B 可控性** | ✅ 已完成 | 单条改分类、单条「本次不移动」 |
| **C 可靠性与平台** | ⏳ 未开始 | 云盘占位、错误归类、可选备份 |
| **D 体验打磨** | ⏳ 未开始 | 整理阶段进度条、首次引导、快捷键等 |
| **E 进阶** | ⏳ 长期 | 监视文件夹、规则导入导出、重复检测等 |

**下一优先建议**：阶段 C（错误提示与平台体验）或 D（整理进度 / 引导），按产品需要择一推进。

---

## 阶段 A：信任与范围（已完成）

- [x] **部分成功不回滚**：单项失败仅记录，成功项保留（已实现）
- [x] **整理后手动撤销**：历史记录 + 整理页横幅一键撤销（已实现）
- [x] **扫描范围说明**：界面明确「默认仅当前目录下一层」；可选「包含子文件夹」递归扫描文件
- [x] **排除规则**：可配置路径/名称片段（如 `node_modules`、`.git`），扫描时跳过
- [x] **执行前预览（Dry-run）**：仅生成 `源路径 → 目标路径` 列表，不写入磁盘；确认后再执行

## 阶段 B：可控性（已完成）

- [x] **扫描结果可编辑**：单条修改目标分类（下拉选择）；执行时使用用户最终分类
- [x] **单条跳过**：勾选「本次不移动」（可选，依赖列表 UI）

## 阶段 C：可靠性与平台

- [ ] **云盘占位文件**：OneDrive 等「仅联机」文件失败时的明确提示
- [ ] **长路径 / 占用**：错误信息归类（权限、被占用、路径非法）
- [ ] **可选备份**：移动前复制到备份目录或回收站策略（高级）

## 阶段 D：体验打磨

- [ ] **整理阶段进度**：移动大量文件时显示进度（当前仅有扫描阶段的进度与「整理中」状态）
- [ ] **首次引导**：空状态说明流程：选目录 → 扫描 → 预览（可选）→ 执行
- [ ] **快捷键**：如 Ctrl+Enter 执行整理（可选）

## 阶段 E：进阶（长期）

- [ ] **定时 / 监视文件夹**
- [ ] **规则导入导出**（分类模板）
- [ ] **重复文件检测**
